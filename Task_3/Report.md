# Task 3 - Data App (Streamlit)

## Задача

Побудувати інтерактивний веб-додаток з двома розділами:
- **Моніторинг метрик** - динаміка email-метрик з виявленням аномалій і AI summary
- **A/B аналіз** - статистичне порівняння тестових груп з AI рекомендацією

## Методологія

**Стек:**
- Streamlit - веб-фреймворк для data apps
- Pandas - обробка даних
- Plotly - інтерактивні графіки
- Scipy (chi2_contingency) - статистичні тести
- Google Gemini API - AI summary і рекомендації

**Розділ 1 - Моніторинг:**
- Фільтри по датах і типу листа в sidebar
- KPI картки: відправлено, read rate, click rate, credits rate
- Інтерактивний графік динаміки з вибором метрики
- Виявлення аномалій: дні де метрика падає нижче 80% від середнього - підсвічуються червоним хрестом
- AI summary через Gemini API - генерує текстовий аналіз обраного періоду

**Розділ 2 - A/B аналіз:**
- Вибір тесту (GROUP_1 - GROUP_4)
- Таблиця метрик Test vs Control
- Chi-square тест для кожної метрики з lift % і p-value
- Heatmap lift % (зелений = Test краще, червоний = гірше)
- AI рекомендація від Gemini: запускати зміну чи ні


## Результати

### Моніторинг метрик

- Виявлено 1 аномальний день (10.10.2024) - read rate впав до ~10% vs середнє 16.58%
- Фільтри по даті і типу листа працюють в реальному часі
- AI summary генерується на основі агрегованих метрик обраного періоду
