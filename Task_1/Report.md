# Task 1 - SQL Queries

## Задача

Написати три SQL запити до таблиці `user_sessions` з полями `user_id`, `first_ts` (unix timestamp), `source`, `pages`.


## Запит 1 - Day-1 Retention (LT1) по датах реєстрації

### Задача
1. Ретеншен LT1 по днях (який відсоток від користувачів залогінилося на наступний день після реєстрації)

### Методологія
- Дата реєстрації = мінімальний `first_ts` по кожному `user_id`, конвертований з unix timestamp у дату
- Повернення = наявність сесії рівно на наступний день після реєстрації
- LEFT JOIN зберігає всіх зареєстрованих юзерів, навіть тих хто не повернувся (інакше retention завжди дорівнював би 100%)

## Запит 2 - Різниця між сторінками юзера і середнім по source

### Задача
2. Вивести різницю кількості переглянутих сторінок кожного користувача і середнього значення по source.

### Методологія
- `SUM(pages)` агрегує всі сторінки юзера по конкретному source
- Window function `AVG() OVER (PARTITION BY source)` рахує середнє по групі без колапсування рядків
- Різниця показує наскільки юзер відхиляється від середнього свого каналу

### Перевірка правильності
Середнє значення `diff_from_source_avg` по кожному source має дорівнювати ~0 (математична властивість відхилення від середнього). Результат перевірки:

## Запит 3 - Структура сторінок в останній день для inactive юзерів (3+ дні)

### Задача
3. Вивести структуру кількості переглянутих сторінок в останній день логіну для користувачів, котрі не заходили 3 і більше днів на цей момент.

### Методологія
- `user_max_date` CTE: знаходимо останній день логіну кожного юзера через `GROUP BY`
- `last_day_pages` CTE: джойнимо сесії з останнім днем і рахуємо pages саме в той день
- `reference_date`: максимальна дата в таблиці як точка відліку для "поточного моменту"
- `JULIANDAY()` дозволяє рахувати різницю в днях між двома датами
- Window function `SUM() OVER ()` рахує загальну суму для відсотків без GROUP BY
